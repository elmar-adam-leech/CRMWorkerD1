### Analyze Current State

The repository has a basic authentication system in place: The `users` table (from shared/schema.ts) includes fields for username, hashed password (bcrypt), role, etc. Authentication logic is handled in server/auth-service.ts (likely with bcrypt.compare for password verification) and exposed via endpoints in server/routes.ts (e.g., POST /login). The frontend has a login form (probably in client/components or pages/login.tsx) that submits credentials.

**Strengths**:
- Secure hashing with bcrypt.
- Multi-tenant ready (users tied to tenants via roles or future fields).
- Existing Gmail OAuth as an alternative auth path.

**Weaknesses**:
- No initial users in DB after schema push, leading to "incorrect password" for non-existent users.
- Potential misconfiguration post-Neon switch: Connection in db.ts must use Neon driver; queries may fail if not.
- Error messages are generic (no distinction between "user not found" vs. "wrong password" for security, but could log details server-side).
- No auto-seed on setup or user registration flow yet.

**Opportunities**:
- Enhance with self-registration (for new contractors).
- Add password reset (email via Gmail integration).
- Test auth in isolation with Vitest.

### Plan Modular Architecture

For sign-in, modularity is light: We'll reuse the existing auth service but add a check in seed script for Neon compatibility. No new interfaces needed; focus on ensuring DB connection works with Neon.

- Feature toggle: Not applicable here, but future auth providers (e.g., OAuth-only) could use tenant_configs.
- User story: As a new user, I can seed a default admin to log in initially.

#### Step 2: Update Seed Script for Neon Compatibility
The seed.ts from before should work with Neon (Drizzle is driver-agnostic), but confirm imports.

1. Open file explorer (left sidebar) → go to "server/scripts" folder → click "seed.ts" to open it.
2. Replace the entire content with this exact code block (updates import paths if needed):
   ```typescript
   import { db } from '../db';
   import { users } from '../../shared/schema'; // Adjust if schema.ts
   import { eq } from 'drizzle-orm';
   import bcrypt from 'bcrypt';

   async function seed() {
     try {
       // Check if admin exists
       const existing = await db.select().from(users).where(eq(users.username, 'admin')).limit(1);
       if (existing.length > 0) {
         console.log('Admin user already exists. Skipping creation.');
         return;
       }

       const hashedPassword = await bcrypt.hash('password', 10);
       await db.insert(users).values({
         username: 'admin',
         password: hashedPassword,
         name: 'Admin User',
         email: 'admin@example.com',
         role: 'super_admin',
         createdAt: new Date(),
       });
       console.log('Default admin user created successfully. Username: admin, Password: password');
     } catch (error) {
       console.error('Seed error:', error);
     }
   }

   seed();
   ```
3. Save (Ctrl+S).
4. Expected: File updated. If import errors, check schema.ts location.
